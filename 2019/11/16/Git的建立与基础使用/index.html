Git的建立与基础使用

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。Git 是 [Linus Torvalds](https://baike.baidu.com/item/Linus Torvalds/9336769) 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件 。

Git与GitHub的区别

Git是一个分布式版本控制系统，简单的说其就是一个软件，用于记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的软件。

GitHub 是一一个为用户提供Git服务的网站，简单说就是一一个可以放代码的地方(不过可以放的当然不仅是代码)。GitHub 除了提供管理Git的web界面外，还提供了订阅、关注、讨论组、在线编辑器等丰富的功能。

Git的常用命令

命令 说明
git config –global user.name “用户名” 将用户名添加到Git，安装后git后只需配置一次
git config –global user.email “邮箱” 将用户邮箱添加到Git，安装后git后只需配置一次
git add 文件名称(包括后缀名) 将某个文件从工作区放到暂存区
git add . 将全部文件从工作区放到暂存区
git add 文件名1 文件名2 文件名3 …… 将多个文件从工作区放到暂存区
git commit -m “说明” 将文件从暂存区存放到本地仓库
git status 查询该操作上一步的工作状态（非必要）
git init 建立本地仓库
gitk 图形化界面

Git的版本回退

将文件每次提交到本地仓库时，git仓库都会为文件生成一个commit id，版本回退则是在此基础上进行操作。

命令 说明
git log 日志的详细信息
git log –pretty=online 按一行打出信息，第一行就是commit id
git reflog 简略形式，常用

Git在本地获取到克隆线上库并操作

注意地址：git clone 仓库具体地址.git

命令 说明
git clone 将远程库下载到本地文件夹内，这样，就有了一个全新的本地库，而且，该本地库已经和远程的库建立了连接。克隆过程需要用户名和密码，而且过程较慢，如果遇到克隆不成功的情况，要进行多次尝试，一般都能成功
git push 该命令用于将本地创建的文件上传到远程库中（完成Git工作是一定运行一下）
git pull 该命令用于将远程库中的文件获取到本地库中（开始Git工作是一定运行一下）

Git分支

命令 说明
git branch 查看当前分支
git branch 分支名 创建新的分支
git checkout 分支名 切换到某个分支
git branch -d 分支名 删除分支
git merge 被合并的分支名 当前分支与某分支合并
git checkout -b 创建新分支并切换到该分支

2.从主分支切换到test分支，发现文字发生了变化，这说明分支的操作互不影响。

3.合并分支

4.删除分支，但一定注意必须退出要删除的分支，该分支才能被删除，即删除分支时，分支不能处于工作状态。

5.除此之外，还可以在分支上建立文件夹，执行操作，但注意，执行完工作后一定注意git push，开始工作前一定注意git pull。

Git的冲突解决

原因：有人通过浏览器端改变了该1.txt文件，而自己在开始工作时有没有执行git pull操作，导致文件冲突

解决方案：先执行git pull命令，打开1.txt文件并修改，尤其要删除git添加的标记部分。最后git push。
