Spring框架

一：为什么时Spring

开发企业级的项目是一件非常复杂的工程，随着Spring的出现，会大大降低J2EE企业级开发的复杂度。作为一种开源技术，Spring几乎替代了EJB技术。并且，Spring不仅仅是替代品，其技术范围比EJB更广、更实用。

二：Spring框架介绍

2.1：Spring技术简介

Spring是一个轻量级Java开发框架，最早是由Rod Johnson创建，目的是为了解决企业级应用开发的业务逻辑层和其他各层的耦合关系，它是一个轻量级开源框架。

2.2：Spring体系结构

Spring迄今为止已经有20多个模块，这些模块分布在

核心容器(Core Container)

数据访问/集成(Data Access/Integration)

Web层

AOP(面向切面编程)模块

植入(Instrumentation)模块

消息传输(Messaging)

测试(Test)模块

2.3：Spring模块简介

(1)核心容器
   Spring的核心容器是其他模块建立的基础，由Spring-core、Spring-beans、Spring-context、Spring-context-support和Spring-expression (Spring 表达式语言)等模块组成。
    ●Spring-core 模块:提供了框架的基本组成部分，包括控制反转( Inversion of Control,IoC)和依赖注入(Dependency Injection, DI) 功能。
    ●Spring-beans模块:提供了BeanFactory, 是工厂模式的-个经典实现，Spring将管理对象称为Bean。
    ●Spring-context 模块:建立在Core和Beans模块的基础之上，提供一个框架式的对象访问方式，是访问定义和配置的任何对象的媒介。ApplicationContext 接口是Context模块的焦点。
    ●Spring-context-support 模块:支持整合第三方库到Spring 应用程序上下文，特别是用于高速缓存(EhCache、 JCache) 和任务调度(CommonJ、 Quartz) 的支持。
    ●Spring-expression 模块:提供了强大的表达式语言去支持运行时查询和操作对象图。这是对JSP2.1规范中规定的统一表达式语言 (Unified EL)的扩展。该语言支持设置和获取属性值、属性分配、方法调用、访问数组、集合和索引器的内容、逻辑和算术运算、变量命名以及从Spring的IoC容器中以名称检索对象。它还支持列表投影、选择以及常见的列表聚合。

(2)AOP和Instrumentation
    ●Spring-aop 模块:提供了一个符合AOP要求的面向切面的编程实现，允许定义方法拦截器和切入点将代码按照功能进行分离，以便干净地解耦。 
    ●Sping-aspets 模块:提供了与AspectJ的集成功能，AspectJ是一个功能强大且成熟的AOP框架。  
    ●Spring-instrument 模块:提供了类植入(Instrumentation)支持和类加载器的实现，可以在特定的应用服务器中使用。

(3)消息
    Sping 4.0以后新增了消息(Spring-messageing模块，该模块提供了对消息传递体系结构和协议的支持。

(4)数据访问/集成
   数据访问/集成层由JDBC、ORM、OXM、JMS和事务模块组成。
    ●Spring-jdbe模块:提供了一个JDBC的抽象层，消除了烦琐的JDBC编码和数据库厂商特有的错误代码解析。
    ●Spring-orm 模块:为流行的对象关系映射(Objec-Relational Mapping) API提供复成层，包括JPA和Hibemae使用Sring.om模块可以将这些OR映射框架与Sprna提供的所有其他功能结合使用，例如声明式事务管理功能。
    ●Sring-oxm 模块:提供了一个支持对象XML映射的抽象层实现，例如JAXEB、Castor、 JiBX 和XStream。
    ●Spring-jms 模块(Java Messaging Service):指Java消息传递服务。包含用于生产和使用消息的功能。 自Sping4.1以后，提供了与Spinig msging模块的集成
    ●Sping-tx 模块(事务模块):支持用于实现特殊接口和所有POJO (普通Java对象)类的编程和声明式事物管理。

(5)Web
  Web层由Sping-wcb. Sping-webmvc、Sprig- wbsocket和Portlet 模块组成。
    ●Spring-web 模块:提供了基本的Web开发集成功能，例如多文件上传功能、使用Servlet监听器初始化一个IoC容器以及Web应用上下文。
    ●Spring-webmvc 模块:也称为Web-Servlet 模块，包含用于Web应用程序的Spring MVC和REST Web Services实现。Spring MVC框架提供了领域模型代码和Web表单之间的清晰分离，并写Spring Framework的所有其他功能集成.
    ●Sprig- wbsocket模块: Spring 4.0以后新增的模块，它提供了WebSocket和SockJs的实现。
    ●Portlet:类似于Sevte模块的功能，提供了Portlet环境下的MVC实现。

(6)测试
  Spring-test模块支持使用JUnit或TestNG对Spring组件进行单元测试和集成测试。

三:开发Spring的Hello World程序

3.1:开发Spring程序的步骤

第一步：定义业务接口和业务实现类
第二步：对业务实现类进行配置
第三步：开发客户端程序进行测试

3.2：编写业务接口

package cn;
/**
 * 
 * 定义业务接口
 *
 */
public interface Hello {

    public String SayHello(String name);

}

3.3：编写接口实现类

package cn;

/**
 * 
 * 定义业务接口的实现类
 *
 */

public class HelloImpl implements Hello {

    @Override
    public String SayHello(String name) {
        // TODO Auto-generated method stub
        return "Hello  "+name;
    }

}

3.4：编写测试类

package cn;

/**
 * 定义测试类
 */
import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;

public class TestHello {

public static void main(String[] args) {
    /**
     * Spring框架通过IoC方式输出Hello World
     * 通过配置文件削解了接口和工厂类之间的耦合
     * application为工厂类对象
     * hello为实体类对象
     */
    @SuppressWarnings("resource")
    ApplicationContext application = new ClassPathXmlApplicationContext("applicationContext.xml");

    Hello hello = (Hello) application.getBean("hello");

    String name="world";

    System.out.println(hello.SayHello(name));

    System.out.println("-----------------------");

    new TestHello().Demo();
}

public void Demo() {
    /**
     * 通过传统方式调用，接口和实现类之间存在耦合
     */
    Hello hi = new HelloImpl();

    String name="world";

    System.out.println(hi.SayHello(name));
}

https://baike.so.com/doc/5725915-5938652.html

3.6:代码思考—三种方式的耦合程度比较

耦合性(Coupling)，也叫耦合度，是对模块间关联程度的度量。耦合的强弱取决于模块间接口的复杂性、调用模块的方式以及通过界面传送数据的多少。模块间的耦合度是指模块之间的依赖关系，包括控制关系、调用关系、数据传递关系。模块间联系越多，其耦合性越强，同时表明其独立性越差( 降低耦合性，可以提高其独立性)。软件设计中通常用耦合度和内聚度作为衡量模块独立程度的标准。划分模块的一个准则就是高内聚低耦合。

也就是考虑耦合性要从三个方面考虑：独立性，可扩展性，重用性

####（1）:面向过程（传统方式）

为解决问题而解决问题，所有的代码在一个类中，代码之间环环相扣。代码独立性低，可扩展性低，重用性低。耦合性较高

举个例子：求长方形的周长和面积
class 长方形{
int height,width;
   public int 面积（int height,int width）{
     Return height*width;
} 
   public int 周长（int height,int width）{
     Return (height+width)*2;
} 
 Public static void main(String[] args){
       System.out.println(new 面积())
       System.out.println(new 周长())
     }   }

###（2）：面向对象

面向对象三大特性为：封装，继承，多态
封装，体现在将对象的方法和属性私有化，并为外界提供公共的访问方法，保护了代码的隐私性。但若把其当做一个模块来看，耦合性较高

继承，在于子类继承了父类，若把子类，父类看做两个单独的模块，则由于子类继承了父类，必然继承了父类的方法及属性，两个模块的独立性较低。但从子类来看，其又可以重写父类方法，也可以添加方法，可扩展性又比较高。因此继承的耦合性还是比较高的。

多态，主要是接口和抽象类的应用，大大降低了耦合度，但接口和实现类之间仍存在耦合。

###（3）：面向接口

在一个面向对象的系统中，系统的各种功能是由许许多多的不同对象协作完成的。在这种情况下，各个对象内部是如何实现自己的,对系统设计人员来讲就不那么重要了;而各个对象之间的协作关系则成为系统设计的关键。小到不同类之间的通信，大到各模块之间的交互，在系统设计之初都是要着重考虑的，这也是系统设计的主要工作内容。面向接口编程就是指按照这种思想来编程。

https://baike.so.com/doc/2320996-2455003.html

在七种耦合度中，接口的耦合程度属于数据耦合(Data Coupling)，耦合度已经很低了，仅仅是接口和实现类之间存在耦合。

3.7：曙光—IoC工厂模式

从3.5中我们得知耦合度低的好处，在3.6中我们又看到，三种编程模式或多或少都会存在耦合，那么有没有一种模式可以完全消除耦合呢？有，IoC工厂模式！
三种模式中，面向接口是耦合度最低的开发模式，但其存在接口和实现类的耦合(UserDao UserDaoImpl)。以此为基点，新建一个工厂类BeanFactory，让接口和实现类之间通过BeanFactory进行连接(UserDao BeanFactory UserDaoImpl),
在传统BeanFactory中：
定义一个静态方法，返回一个实现类，在测试类中创建工厂类对象，接收该实现类。

class BeanFactory{
   public static UserDao getUserDao(){
      return UserDaoImpl;
}     
public static Customer getCustomer(){
      return CustomerImpl;
}
 。。。。。。。。。}

虽然消除了接口和实现类之间的耦合，但接口却和工厂类有了耦合，于是，提出了IoC工厂模式，即“工厂”+“反射”+“配置文件”

3.7.2：BeanFactory和ApplicationContext的区别

BeanFacotry是spring中比较原始的Factory。如XMLBeanFactory就是一种典型的BeanFactory。原始的BeanFactory无法支持spring的许多插件，如AOP功能、Web应用等。
ApplicationContext接口,它由BeanFactory接口派生而来，因而提供BeanFactory所有的功能。ApplicationContext以一种更向面向框架的方式工作以及对上下文进行分层和实现继承，ApplicationContext包还提供了以下的功能：

• MessageSource, 提供国际化的消息访问

• 资源访问，如URL和文件

• 事件传播
• 载入多个（有继承关系）上下文 ，使得每一个上下文都专注于 一个特定的层次，比如应用的web层

因此，现有阶段使用ApplicationContext制造工厂，而不使用BeanFactory

3.7.3：注意

1：IoC可以认为是一种全新的设计模式，可以把IoC模式看做是工厂模式的升华

2：当前比较知名的IoC容器有:Pico Container、Avalon 、Spring、
JBoss、HiveMind、EJB等。因此，IoC并不是Spring独有的。

3：IoC是基于面向接口展开的。

4：IoC是Spring的核心技术之一，Spring有两大核心技术：IoC/DI、AOP，这也是为什么要学习IoC。

5：IoC是一个很大的概念，可以用不同的方式实现，其主要方式有两种，“依赖查找”和“依赖注入”

6：使用IoC之后，意味着你的实现类交给了Spring框架进行管理，那么，你在实现类中进行的一系列的操作均可以在配置文件中进行设置，而无需改动实现类的代码。在DI中会有详细讲解。

7：ApplicationContext的连个实现类
